## 1. Что такое Claude Design

Claude Design — продукт Anthropic Labs, который превращает текстовое описание в визуальный артефакт: презентацию, интерактивный прототип, лендинг, инфографику. Запущен 17 апреля 2026 года. Работает на модели Claude Opus 4.7. Доступен по адресу claude.ai/design только в браузере — отдельного приложения для Mac или мобильного нет.

**Ключевая идея:** не заменить Figma или Canva, а закрыть стадию «от идеи до первого драфта». После Claude Design проект можно дошлифовать в Canva, перенести в Figma или отдать в разработку через Claude Code.

Выходит не статичная картинка, а живой артефакт: презентация с редактируемым текстом, прототип с рабочими переходами, лендинг в виде HTML-бандла.

**Для кого (роли на онбординге):**
- Design — дизайнеры (первый драфт + передача в Figma)
- Engineering — разработчики (макеты + handoff в код)
- Product — продакты (прототипы, wireframes, user-flow)
- Sales — продажники
- Data — аналитики (дашборды, визуализации)
- Marketing — маркетологи (креативы, карусели, лендинги)
- Other — остальное

## 2. Доступ и тарифы

Claude Design включён без доплаты во все платные подписки Claude, расходует существующую квоту:

| Тариф | Цена | Для чего |
|---|---|---|
| Pro | ≈$20/мес | Попробовать, пара проектов в неделю |
| Max | ≈$100–200/мес | Регулярная работа |
| Team | от $25/место | Корпоративная команда |
| Enterprise | договорной | Большие организации |

**Бесплатный тариф не даёт доступа во время research preview.**

**Как попасть:**
1. claude.ai → в сайдбаре пункт Design
2. Если пункта нет — выйти из аккаунта и войти заново или подождать (раскатка постепенная)
3. Для Enterprise — включает админ через org settings

**Токеномика (важно):**
- 1 дизайн-система + 1 прототип лендинга = 65% недельной квоты Pro
- 2 полных дизайн-сессии = 58% недельной Pro
- Сложный слайд-дек = 5–7-минутная reasoning-сессия Opus 4.7

Для регулярной работы реалистичны Max или Team. Pro — чтобы попробовать.

**Общие ограничения:**
- Системный интерфейс: английский, русский, испанский и ещё несколько языков
- Контент можно писать на любом языке
- Нет публичного API на момент запуска
- Нет интеграции с Zapier/Make

## 3. Интерфейс

### Главный экран — две зоны + верхний тулбар

**Левая панель (чат + контекст):**
- Окно промпта
- 4 кнопки прикрепления контекста:
  - Design System — подцепить дизайн-систему
  - Add screenshot — прикрепить скриншот
  - Attach codebase — подцепить репозиторий
  - Drag in a Figma file — загрузить .fig или Figma-URL
- История чата по проекту (сохраняется, уезжает в handoff-бандл)

**Правая панель (холст):**
- Рабочая область с результатом
- Вкладки: HTML assets / UI kits / Uploads
- В прототипах можно кликать, переключаться, листать

**Верхний тулбар — четыре режима редактирования:**
| Режим | Что делает |
|---|---|
| Tweaks | Ад-хок ползунки под проект (spacing, цвет, шрифт, радиус, скорость) |
| Comment | Пин-комментарии к конкретным элементам |
| Edit | Прямая правка текста/картинок на холсте (как в Word) |
| Draw | Рисование поверх: скетч, стрелки, пометки, новые элементы |

Плюс кнопки Export и Hand off to Claude Code.

### Виды проектов (выбор при создании)

| Таб | Параметры | Для чего |
|---|---|---|
| Prototype | Fidelity (Wireframe / High) | Интерактивные макеты |
| Slide deck | Speaker notes (on/off) | Презентации, pitch-деки |
| From template | Выбор шаблона | Быстрый старт |
| Other | Произвольный промпт | Лендинги, one-pager, инфографика |

Marketing collateral, социальные посты, обложки — через таб Other.

### Design files — файловый менеджер

Вкладка Design files — встроенное хранилище проекта:
- Загруженные ассеты (логотипы, брендбуки, PDF-спеки)
- Сгенерированные HTML-ассеты и UI-киты
- Skills (см. ниже)

### Skills внутри Claude Design

Сохранённые «знания» Claude в рамках проекта: brand voice, структура лендинга, типовые CTA. Открываются по кнопке Open в Design files. Полезны для повторяющейся работы.

### Горячие клавиши

| Клавиша | Действие |
|---|---|
| ⌘K / Ctrl+K | Глобальный поиск |
| Ctrl/Cmd + G | Открыть чат в режиме Draw |
| Escape | Выйти из full-screen |
| ⌘S / Ctrl+S | Ручное сохранение |

## 4. Входные данные и контекст

Claude Design принимает 11 типов входов. Чем больше контекста — тем точнее первый драфт.

| Тип | Как подать | Когда |
|---|---|---|
| Текстовый промпт | В чат | Короткое ТЗ |
| Ссылка на сайт | URL в промпте | «В таком же стиле» |
| Скриншот-референс | Add screenshot | «Хочу похоже на это» |
| Фото-референс | Drag-n-drop | Настроение, mood |
| Рукописный скетч | Фото наброска | Структура страницы |
| PDF | Загрузка | Длинное ТЗ |
| DOCX | Загрузка | Текст от копирайтера |
| PPTX | Загрузка | Перенести старую презу |
| XLSX / CSV | Загрузка | Данные для графиков |
| GitHub-репо | URL + авторизация | Авто-дизайн-система |
| Figma-файл | URL или .fig | Перенос библиотеки |

### Дизайн-система: настройка (главный ритуал)

**Правило:** настроить один раз, пользоваться годами.

1. На онбординге или в настройках проекта — Set up design system
2. Подать источник:
   - Ссылку на сайт — Claude через DOM тянет цвета, шрифты, структуру
   - GitHub-репо — читает Tailwind-конфиг, CSS-переменные, компоненты
   - Figma-файл — извлекает библиотеку стилей
   - PDF-брендбук — парсит гайдлайны
3. Claude формирует: палитра, типографика, отступы, радиусы, компоненты
4. Новые проекты наследуют автоматически

**Multiple design systems** (Team/Enterprise) — несколько систем в одной области (B2B, B2C, личный блог). Переключение в настройках проекта.

### Web Capture Tool

Указываешь URL живого сайта, Claude через DOM-инспекцию (не скриншот!) вытаскивает структуру, цвета, типографику. Он видит настоящий HTML/CSS.

**Применение:**
- «Сделай лендинг как у [конкурент], но для нашего продукта»
- «Возьми футер с этого сайта и примени стиль к моему»
- «Нашему сайту 5 лет — перерисуй в современный стиль»

## 5. Промпты и генерация

### Главный принцип

**Не описывай дизайн — описывай задачу.**

Плохо: `Сделай синий градиент с закруглёнными кнопками и fading-эффектом в hero.`

Хорошо: `Лендинг онлайн-курса по фотографии для начинающих. Аудитория — студенты и молодые специалисты. Тон — тёплый, без давления. Главная цель — записать на бесплатный пробный урок.`

### Формула промпта (4 обязательных элемента)

1. **Формат** — что делаем (лендинг / 10 слайдов / квадратный пост 1080×1080 / экран приложения)
2. **Аудитория** — кто увидит (возраст, контекст, уровень)
3. **Задача** — что должно произойти после (записались, купили, поняли, запомнили)
4. **Тон** — как звучит (уверенный, спокойный, резкий, дружеский, технический)

**Опциональные усилители:**
- Референс (URL или скриншот)
- Ограничения («без стоковых фото», «не использовать градиенты»)
- Длина («не больше трёх блоков», «ровно 5 слайдов»)

### Универсальный шаблон (работает в 90% случаев)

```
[Формат] для [аудитория].
Задача: [что должно произойти].
Тон: [какой].
Ключевые блоки: [список].
Бренд: [URL или "как в моей дизайн-системе"].
Ограничения: [что не делать].
```

### Правила итерации

**Сначала структура, потом эстетика:**
1. Сначала дожми контент и структуру
2. Потом крути цвета, шрифты, радиусы

**Правило уровней:**
- Структура и логика → чат
- Точечные изменения текста → режим Edit
- Просьба к элементу → режим Comment
- Масштабные параметры → режим Tweaks
- Добавить элемент/показать идею → режим Draw

**3 варианта, а не 3 итерации:**
Первый результат не то — просить сразу три варианта в разных направлениях. Сравнивать проще.

## 6. Режимы редактирования

### Tweaks — ад-хок ползунки

Claude генерирует ползунки **под проект индивидуально**, не фиксированный набор:
- Для презентации: плотность слайдов, контрастность, типографика, акцентный цвет
- Для лендинга: радиус скруглений, густота отступов, интенсивность свечения кнопок
- Для 3D-прототипа: скорость вращения, интенсивность свечения

Работает глобально, а не по элементу.

### Comment — пин на элемент

Пин + текст:
- «Этот заголовок звучит казённо — перепиши человечнее»
- «Замени иконку на более минималистичную»
- «Здесь нужно фото, а не иллюстрация»

**Баг:** inline-комменты иногда пропадают. Критичное дублируй в чат.

### Edit — прямая правка

Как в Word: кликнул — переписал. Используй для:
- Опечаток
- Персонализации («Иван Иванов» → «Анна Смирнова»)
- Точечных замен

Не используй для структурных правок — их проще через чат.

### Draw — рисование поверх

Стрелки, пометки, новые блоки:
- Показать куда переместить элемент
- Нарисовать блок и попросить Claude превратить скетч в компонент
- Выделить что убрать

Внутри Draw: **Ctrl+G / Cmd+G** открывает чат — «преврати этот скетч в реальный блок».

### Правила правок

1. **Одна правка — одна команда.** «Сделай hero светлее И подвинь CTA И замени иконку» — Claude зацепит одно, остальное проиграет. Лучше три шага подряд.
2. **Конкретика побеждает.** «Сделай лучше» — бесполезно. «Замени серый фон #F5F5F5 на тёплый бежевый #F2EAD3» — работает.
3. **Референс — друг.** «Сделай как на этом скриншоте [ссылка]» лучше любого описания.

## 7. Экспорт

7 форматов. Выбор зависит от того, что делаем с файлом дальше.

| Формат | Редактируемость | Когда |
|---|---|---|
| Canva | Всё — в Canva-редакторе | Команда правит в Canva |
| PowerPoint .pptx | Текст в текстовых полях (если выбрать editable text) | Клиент правит в PowerPoint |
| PDF | Ничего, но векторное качество | Финал, печать |
| Standalone HTML | Всё — код в ZIP | Публикация лендинга |
| ZIP archive | Сырые исходники | Бэкап, перенос разработчику |
| Internal URL | Полностью — ссылка внутри орга | Быстро показать коллеге |
| Claude Code Bundle | Токены, компоненты, README | Handoff в код |

### Особенности

- **PPTX:** галочка editable text обязательна, иначе получишь плоские картинки
- **PDF:** вектор, не растр — не пикселится
- **HTML-экспорт:** чистый статичный сайт, можно на Vercel/Netlify
- **Internal URL:** доступен только внутри Claude-организации

### Куда что выгружать

| Сценарий | Формат |
|---|---|
| Питч-дек инвестору | PDF + PPTX |
| Маркетологам на доработку | Canva |
| Разработчикам | Claude Code Bundle или HTML |
| В социальные сети | PDF → конвертация в PNG |
| Печать А4/визитка | PDF |
| Внутренний обзор | Internal URL |
| Перенос в Figma | HTML (через плагин) |

## 8. Handoff в Claude Code

Главная суперсила экосистемы: закрытый цикл «идея → дизайн → код».

### Как работает

1. Export → Hand off to Claude Code
2. Выбор агента:
   - Local coding agent — Claude Code в CLI или VS Code локально
   - Claude Code Web — облачный агент в Claude Cowork
3. Claude Design генерирует terminal-команду:
```
claude-code implement https://design.anthropic.com/bundle/xxxxx \
  --instruction "Build the landing page exactly as designed" \
  --file src/pages/landing.tsx
```
4. Копируешь команду
5. Вставляешь в открытую сессию Claude Code

### Что в bundle

- **Components** — описания компонентов и их границы
- **Design tokens** — цвета, шрифты, отступы, радиусы как переменные
- **Copy** — весь текст
- **Interaction notes** — поведение при hover/click/focus
- **Design files** — оригинальные исходники
- **Chat history** — вся переписка (контекст решений)
- **README** — инструкция для Claude Code

### Что нужно для работы

- Установленный Claude Code последней версии
- Открытый репозиторий
- Дизайн-система подцеплена к этому репо (код совпадёт со стеком и конвенциями)
- Совпадение стека: если Vue — указать явно `--stack vue3+ts`

### Для чего полезно

- Промо-страницы под события, конференции, запуски
- Мокапы новых экранов → рабочий прототип
- Внутренние инструменты компании (админки, формы, дашборды)
- A/B-версии одного экрана

### Где не выручает

- Сложная бизнес-логика (валидация форм, интеграция с платёжкой — проверять руками)
- Кастомные анимации уровня Lottie/After Effects
- Компоненты с тонкой доступностью (сложные ARIA-атрибуты)

### Альтернатива: Figma MCP

Отдельный плагин, Claude Code читает Figma-файл напрямую. Код из Claude Code можно превратить в редактируемый Figma-дизайн и обратно.

## 9. Frontier Design

Интерактивные код-пауэрд компоненты — живые элементы внутри макета.

| Компонент | Что делает |
|---|---|
| 3D-элементы | Вращающиеся глобусы, спирали, 3D-графики |
| Шейдеры | Анимированные фоны, градиенты с физикой |
| Voice-модули | Встроенный голосовой интерфейс |
| Видео | Встроенные видео-плееры |
| AI-компоненты | Мини-чатбот внутри макета, генеративные виджеты |

**Когда нужно:**
- Pitch-дек технологическим инвесторам
- Лендинг продукта, где интерактив — часть сообщения
- Презентации на AI/web3-конференциях

**Когда избыточно:**
- Классический corporate pitch
- Маркетинговый лендинг для массовой аудитории
- Инструкция или one-pager

**Известный баг:** 3D-глобус иногда рендерится криво — проверяй визуально.

## 10. Команда и организация

### Team plan

- Общая рабочая область
- Shared design systems (несколько систем)
- История проектов команды
- Асинхронные комментарии в проектах

### Enterprise plan

- Выключен по дефолту, админ включает
- RBAC (Role-Based Access Control) по департаментам
- Admin guide: support.claude.com/en/articles/14604406
- Централизованный биллинг и логи
- SSO

### Чего нет на момент запуска

- **Real-time co-editing** — три дизайнера в одном файле одновременно нельзя
- **Version history** в духе Google Docs — только чат-история
- **Fine-grained permissions** на уровне отдельных файлов

## 11. 15 воркфлоу по ролям

### Предприниматель: pitch-дек инвестору

```
Pitch-дек для венчурного инвестора seed-стадии. 12 слайдов.
Продукт — [описание].
Аудитория — фонд, который инвестирует в B2B SaaS.
Структура: обложка, проблема (одна фраза + цифра масштаба), рынок в числах,
наше решение, продукт в действии (скриншот/макет), бизнес-модель,
трекшен (MRR, рост MoM, retention), команда (3 человека с ролями),
конкуренты (позиционирующая матрица 2×2), план на 12 месяцев,
сумма раунда и использование средств, контакты.
Тон: уверенный, без маркетинговой воды. На каждом слайде — один главный тезис.
Бренд: [URL сайта или "как в моей дизайн-системе"].
```
Экспорт: PDF + PPTX. Совет: первые 3 слайда доведи до идеала.

### Предприниматель: лендинг продукта

```
Лендинг продукта [название].
Что делает: [одна фраза].
Аудитория: [кто, возраст, технический уровень].
Цель: оставить email для доступа к бета-версии.
Структура: hero с ключевой ценностью + CTA, как работает в 3 шага,
3 главных фичи (иконка + заголовок + описание),
социальное доказательство (3 отзыва с фото и ролями),
FAQ (6 вопросов), финальный CTA, футер.
Стиль: минималистичный, светлый, один акцентный цвет.
Без стоковых фото людей. Иконки — линейные.
```
Экспорт: HTML + Internal URL.

### Маркетолог: рекламные креативы для A/B

```
8 вариантов креативов для Яндекс.Директ и VK.
Продукт: [описание]. Сегмент ЦА: [кто].
4 × 1080×1080 (квадрат) + 4 × 1200×628 (баннер).
4 архетипа сообщения (по 2 формата):
1. Страх упущенной выгоды
2. Социальное доказательство
```

## 12. Частые ошибки

| Ошибка | Как избежать |
|---|---|
| Не похоже на бренд | Настрой дизайн-систему через GitHub/Figma/URL |
| Тексты канцелярские | Всегда указывай тон и дай пример языка |
| Сгенерил стоковые иллюстрации | В промпте: «без стоков, только абстракции» |
| Токены кончились посреди проекта | Делай секциями по 3–4 слайда |
| Правки через чат игнорятся для элемента | Используй Comment — пин на элемент |
| PPTX открылся плоскими картинками | Галочка editable PPTX на экспорте |
| Handoff не работает | Обнови Claude Code до последней версии |
| Inline-коммент пропал | Дублируй критическое в чате |
| Нет empty/error/loading states | Проси явно: «добавь empty, loading, error» |
| 3D-элемент рендерится криво | Проверяй визуально, переразгенеривать |
| Всё выглядит генеративно-AI | Задай тон, ограничения, референсы |

## 13. Лайфхаки

1. **Шаблоны через клонирование.** Сделал идеальную обложку — клонируй проект, меняй одно поле, стиль держится.

2. **Серийные промпты для A/B.** Один базовый промпт, сверху «сделай то же, но с фокусом на [X]». 5 вариантов за 5 раз вместо 5 итераций.

3. **Загрузи свои тексты перед генерацией.** Есть стиль (посты, статьи) — скорми 2–3 текста. Claude подхватит манеру.

4. **«Покажи в трёх настроениях».** Вместо «сделай» — «сделай в трёх версиях: тёплая, строгая, дерзкая».

5. **Web capture для старта.** У конкурента красивый лендинг — скорми URL, «такую же структуру, но для нашего продукта».

6. **Дизайн-система — инвестиция 15 минут.** Один раз настроил — годами пользуешься.

7. **Skills для повторяющегося.** Сохрани типовой CTA, brand voice, любимую структуру.

8. **Дроби большие проекты.** Презентация на 25 слайдов? По 5 за заход. Качество выше, токенов меньше.

9. **Сохраняй рабочие промпты.** Хороший промпт — в отдельный файл. Персональная библиотека.

10. **PPTX — дефолт для презентаций.** Даже если сейчас нужен PDF — экспортируй оба.

11. **Draw + Ctrl+G для быстрых идей.** Нарисовал стрелку — сразу в чат: «сделай кнопку куда указывает».

12. **Chat history в handoff.** Bundle включает переписку — разработчик поймёт контекст решений.

13. **Multiple design systems для разных продуктов.** B2B, B2C, личный блог — каждому своя.

14. **Internal URL на созвонах** удобнее PDF.

15. **Явные ограничения.** «Не используй градиенты. Без стоков. Без слова "инновационный".» Меньше плохих итераций.

## 14. Ограничения

### Не умеет

- **Фотореалистичные изображения** — рисует SVG, фото грузи сам
- **AI-арт** — никаких Midjourney/DALL-E внутри
- **Сложные анимации** — Lottie-уровня нет
- **Auto-layout компоненты** — уровень Figma здесь ниже
- **Real-time co-editing** — только асинхронно
- **Pixel-perfect контроль** — инструмент идеи, не финального продакшена

### Делает нестабильно

- Inline-комменты теряются — дублируй в чат
- SVG-пути иногда с артефактами — проверяй
- Dark-mode palette не везде распространяется
- Empty/error/loading states пропускаются — проси явно
- Кроссбраузерность HTML — тестируй в Safari и Chrome

### Инфраструктурные

- Только в браузере (нет Mac/Windows/iOS/Android-приложений)
- Нет публичного API
- Нет интеграций Zapier/Make
- Нет экспорта в Sketch/Adobe XD
- Research preview — может сломаться

### Когда не брать Claude Design

- Pixel-perfect macOS/iOS UI → Figma
- Живое co-editing → Figma
- Сложная иллюстрация под печать → Illustrator
- Фоторедакция → Photoshop
- Типовые шаблоны «быстро и просто» → Canva

## 15. Сравнение с альтернативами

### vs Figma

| | Claude Design | Figma |
|---|---|---|
| Порог входа | Минимальный | Недели |
| Управление | Словами | Мышкой |
| Первый драфт | 5–10 мин | 1–3 часа |
| Pixel-perfect | Сложно | Стандарт |
| Co-editing | Нет | Да |
| Auto-layout | Ограниченно | Полноценно |
| Handoff | Claude Code | Figma MCP / Dev Mode |

**Вывод:** Claude Design — быстрый старт, Figma — финализация и командная работа.

### vs Canva

| | Claude Design | Canva |
|---|---|---|
| Старт с нуля | По промпту | Выбор из шаблона |
| Кастомизация | Чат + Tweaks | Drag-n-drop |
| Кликабельные прототипы | Да | Нет |
| Handoff в код | Да | Нет |
| Шаблоны соцсетей | Генерит | Библиотека тысяч |
| 3D и интерактив | Да | Нет |

**Вывод:** Canva для шаблонов, Claude Design для уникального и для разработки.

### vs Adobe

| | Claude Design | Adobe |
|---|---|---|
| Печать / препресс | Нет | Стандарт |
| Сложная иллюстрация | SVG-уровень | Безграничная |
| Фоторедакция | Нет | Photoshop |
| Длинные документы | Сложно | InDesign |
| Скорость первого драфта | 5–10 мин | Часы-дни |

### Решающая схема

```
Нужен визуал
 │
 ├── Финал для печати/ретуши? ── да ──→ Adobe
 ├── Нужен шаблон и красиво? ── да ──→ Canva
 ├── Команда дизайнеров с co-editing? ── да ──→ Figma
 ├── Первый драфт, быстро, словами? ── да ──→ Claude Design
 └── Прототип → сразу в прод-код? ── да ──→ Claude Design + Claude Code
```
