# Шаблон: боевая карта конкурентов (battlecard)

> **Что это:** живой документ-скелет конкурентной разведки по вашему продукту `{{МОЙ_ПРОДУКТ}}`. Заполняете и обновляете после каждого нового разбора (демо конкурента, разбор сайта, разговор с экс-клиентом, тайное интервью). Это ваш единый источник правды по позиционированию на рынке.
>
> **Методология:** взгляд через [AJTBD-линзу](../methodology/04-ajtbd-lens.md). Главная идея: конкурент — это **не только другой похожий продукт**, а **всё, что клиент нанимает на ту же задачу (Job)**. Сюда входят и таблицы Excel, и ручная работа сотрудника, и вариант «ничего не делать».
>
> **Как читать таблицы:** строка = один конкурент или одна альтернатива. «Угроза 1-5» = насколько он опасен для вашего сегмента (1 — почти не пересекаетесь, 5 — лобовое столкновение). «Свежесть» = когда последний раз обновляли эту строку.

---

## Как пользоваться этим шаблоном

1. Скопируйте файл к себе и переименуйте под свой продукт.
2. Замените плейсхолдеры на свои значения:
   - `{{МОЙ_ПРОДУКТ}}` — название вашего продукта.
   - `{{НИША}}` — рынок/сфера, где вы работаете.
   - `{{КОНКУРЕНТ}}` — имя конкретного конкурента.
   - `{{САЙТ_URL}}` — ссылка на сайт конкурента.
3. В каждой таблице оставлена одна пример-строка («Конкурент А») и пустые строки — заполняйте по мере разборов.
4. Детальные разборы каждого конкурента держите отдельными файлами (см. [шаблон разбора источника](source-breakdown.md)) и ссылайтесь на них из колонки «Детали».

> Прямые конкуренты — продукты, которые клиент явно сравнивает с вами. Чем больше колонок заполнено, тем полезнее карта. Пустые ячейки (`—`) — это нормально на старте: помечайте их и закрывайте по мере сбора данных.

**Подсказка по колонкам:**
- **Core Job** — пишите их формулировку, не свою. Сравнение чужих формулировок Job само по себе подсказывает, как они себя позиционируют.
- **Свежесть с пометкой `(lite)`** ставьте, если строка собрана быстро по сайту, без глубокого разбора. `(demo)` — после живого демо. `(2-е лицо)` — со слов кого-то третьего, проверять отдельно.

## II. Невидимые конкуренты (главные — по AJTBD)

> Это часто настоящие конкуренты вашего продукта: туда уходит бюджет и внимание клиента, когда он считает, что «и так справляется». Не игнорируйте их — именно здесь чаще всего проигрывают сделки.

| Альтернатива | Что закрывает | Почему её выбирают | Где она ломается (ваш заход) | Угроза 1-5 |
|--------------|---------------|--------------------|------------------------------|------------|
| Таблица Excel / Google Sheets _(пример)_ | минимальный контроль/учёт своими руками | бесплатно, привычно, гибко | не масштабируется на большую команду, нет аналитики паттернов | 3 |
| Ручная работа сотрудника _(пример)_ | задача закрывается «по верхам» вручную | доверие к человеку, нет порога входа | выгорание, выборочность, субъективность, не масштабируется | 4 |
| «Ничего не делать» (статус-кво) _(пример)_ | — | «и так работает», «сейчас не время» | триггер появляется при провале/инциденте/росте — ловите этот момент | — |
| Разовая внешняя услуга/консалтинг _(пример)_ | разовый результат → отчёт | эксперт со стороны, не нужно внедрять | дорого, разово, нет постоянного цикла и накопления данных | — |
|  |  |  |  |  |
|  |  |  |  |  |

**Подсказка:** чтобы найти невидимых конкурентов, спросите клиента не «какие продукты вы смотрели», а «что вы делаете с этой задачей сейчас». Ответ почти всегда — один из вариантов выше.

## III. Switching Forces — ваша сводка

> AJTBD выделяет 4 силы, которые действуют на клиента в момент решения переключиться. Две тянут к новому решению (к вам), две держат на старом. Заполняйте по мере накопления данных из интервью, демо и разборов. Подробнее — в [AJTBD-линзе](../methodology/04-ajtbd-lens.md).

### ➕ Силы ЗА переключение к вам (что можно усилить)

- **Доп. ценность, которую видит клиент:** _что нового он получает, чего нет в старом решении._ Пример: «вижу всю задачу целиком, а не отдельный фрагмент» / «нахожу проблемы, которые раньше тихо терялись».
- **Проблемы со старым решением, которые цепляют:** _что в текущем способе раздражает настолько, что он готов двигаться._ Пример: ручной способ «съедает кучу времени»; после сокращения штата образовалась дыра в контроле.

### ➖ Силы ПРОТИВ переключения (что нужно снимать)

- **Барьеры (на что вы можете влиять):** _конкретные страхи и трения перед покупкой._ Частые примеры: страх передавать чувствительные данные до выбора подрядчика (NDA-трение); паралич выбора — «несколько похожих предложений с одинаковым ценником, сложно выбрать». Снимаются чётким one-liner отличия + лёгким и безопасным быстрым proof.
- **Привычка / инерция (тормозит сильнее всего):** _процессная инерция, которую трудно сдвинуть._ Пример: «сначала посмотрим всех, потом будем выбирать»; нет ощущения срочности. Лечится тем, что вы помогаете клиенту посчитать **цену бездействия** в его деньгах.

## IV. Матрица механик создания ценности

> Какие механики создания ценности применяют конкуренты, какие применяете вы, а где **никто** не работает (это и есть ваши возможности для отстройки). Список механик ниже — рабочий минимум; адаптируйте под свою категорию.

| Механика | Применяют (кто и как) | Применяете вы | Гэп (никто не делает) |
|----------|------------------------|---------------|-----------------------|
| Починить боль | Конкурент А: закрывает основную боль клиента | — | — |
| Сделать дешевле | _кто демпингует?_ | — | _свободен ли дешёвый сегмент?_ |
| Сделать быстрее | — | — | — |
| Сделать точнее / надёжнее | — | — | — |
| Снять страх (пилот, гарантия, безопасность данных) | Конкурент А: бесплатный пилот при годовой оплате | — | снять страх за данные на уровне продукта |
| Социальное доказательство | Конкурент А: логотипы известных клиентов | — | — |
| Снизить порог входа | — | — | мгновенный демо «на своих данных» |
| Принести знание / бенчмарки | Конкурент А: отраслевые бенчмарки | — | — |
| Подсветить незаметную проблему | Конкурент А: показывает скрытые риски | — | — |
| Дать новые возможности | — | — | — |
| Выйти на работу уровнем выше | Конкурент А: от «сделать операцию» к «повлиять на результат» | — | — |
| Объединить несколько работ в одном | — | — | — |
| Снять привычку / создать срочность | _кто давит на срочность?_ | — | «цена месяца бездействия» |

**Где ваша возможность:** колонка «Гэп» — это потенциальные направления отстройки. Если механику не делает никто на рынке, а клиенту она нужна — это кандидат на ваше отличие.

> Примеры мульти-доменно: «сделать быстрее» в SaaS — мгновенный онбординг без внедренца; в услугах — результат за день вместо недели; в рознице — выдача заказа за минуту; на производстве — настройка линии без инженера на площадке.

## V. Aha-moments конкурентов (упаковка)

> Что в демо/презентации конкурента вызывает «ого» у клиента. Цель — **скопировать приём подачи, а не саму фичу**. Один и тот же приём часто переносится в любую сферу.

| Конкурент | Aha-moment (что вызывает «ого») | Как они его подают (приём) |
|-----------|--------------------------------|----------------------------|
| **Конкурент А** _(пример)_ | «Показали проблему, которую клиент сам не замечал, → клиент вслух сказал ‘а это сильно’» | Discovery-first: первые минуты демо — про боль клиента, а не про продукт. Активирующее знание показывают на живом примере данных клиента. |
|  |  |  |
|  |  |  |

**Как собирать этот раздел:** на каждом разборе фиксируйте момент, когда у зрителя «загорелись глаза», и отдельно — **механику подачи** (порядок экранов, формулировка, демонстрация «продукт на себе», перевод выгоды в часы/деньги). Готовый разбор таких приёмов — в [examples/packaging-tactics.md](../examples/packaging-tactics.md).

## Ритуал обновления

После каждого нового разбора:

1. Создайте детальный файл по [шаблону разбора источника](source-breakdown.md) в папке конкурента (`<имя-конкурента>/`).
2. **Обновите таблицы этого файла** — одна строка / одна ячейка по итогам.
3. Поставьте дату в поле «Свежесть» у затронутой строки.
4. Если нашли новый приём упаковки → добавьте в раздел V и в [examples/packaging-tactics.md](../examples/packaging-tactics.md).
5. Если нашли новую невидимую альтернативу → добавьте в раздел II.
6. Запишите изменение в Changelog ниже.

## Changelog

> Ведите хронологию: что добавили/обновили и из какого источника. Это защищает карту от «тихого устаревания» — всегда видно, насколько свежи данные.

- **ГГГГ-ММ-ДД** — _краткое описание изменения и источник (например: «добавлен Конкурент А по итогам демо, заполнены разделы I, III, V»)._
- **ГГГГ-ММ-ДД** — создан battlecard из шаблона, скелет таблиц, методологическая база (AJTBD). Данные конкурентов — плейсхолдеры, заполняются по факту разборов.
