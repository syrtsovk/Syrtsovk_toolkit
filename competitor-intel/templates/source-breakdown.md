# Шаблон разбора одного источника

> **Что это:** структурированный шаблон, чтобы разобрать один источник о конкуренте — демо-звонок, лендинг, интервью с бывшим клиентом — через призму AJTBD (Jobs To Be Done). Один источник = один заполненный файл.
> **Зачем:** превратить «посмотрел демо и что-то понял» в конкретные факты по 10 блокам, которые потом складываются в [боевую карточку](battlecard.md).

---

## Как пользоваться

1. Скопируй этот шаблон в файл с понятным именем, например `sources/{{КОНКУРЕНТ}}-demo-2026-01.md` (или `-site`, `-interview` — по типу источника).
2. Заполняй блоки по порядку. Если факта в источнике нет — ставь `—`, не выдумывай.
3. Маркеры на полях:
   - *(без маркера)* — факт прямо из источника (цитата или близкий пересказ).
   - `^[inferred]` — это твой вывод / синтез, в источнике явно не сказано.
   - `^[ambiguous]` — источник неясен или данные противоречат друг другу.
4. Если разбираешь демо или питч по транскрипту — прогони его сначала через [XML-промпт разбора демо](../prompts/demo-analysis-xml-prompt.md), а результат разложи по блокам ниже.
5. Заполнил — обнови сводную [боевую карточку](battlecard.md) одной строкой.

**Плейсхолдеры:** `{{КОНКУРЕНТ}}` — кого разбираем · `{{НИША}}` — рынок/категория · `{{САЙТ_URL}}` — источник · `{{МОЙ_ПРОДУКТ}}` — твой продукт для сравнения.
